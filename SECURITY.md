# Security policy

## Supported release

ThreatScope XDR 1.0.x is the supported maintenance line once v1.0.0 is tagged. Unreleased working trees and older roadmap phases receive fixes only when they are required to prepare or maintain that line.

## Reporting a vulnerability

Use the repository host's private security-advisory mechanism when available. No repository-approved security email address exists, so this project does not invent one. Include the affected version/component, prerequisites, minimal reproduction using only systems you own or are authorized to test, impact, and a suggested mitigation if known. Do not include real passwords, tokens, session/CSRF values, private keys, personal data, malicious live payloads, or third-party targets.

Maintainers should acknowledge receipt and coordinate validation/remediation/disclosure, but no guaranteed response time is promised. Keep details private until a fix and disclosure plan are agreed. Scope includes application/backend/frontend controls, local deployment files, release tooling, and bundled test target. Social engineering, denial of service against other parties, and unauthorized scanning are out of scope.
